# 福井中文社交媒体数据抓取手动运行指南

本文档面向不熟悉编程的 Windows 用户，默认使用 **VS Code + WSL Ubuntu** 运行本项目。请按顺序操作，不要跳过“备份与安全规则”部分。

## 0. 第一次获取代码（只需做一次）

在 VS Code 的 WSL Ubuntu 终端里依次执行：

```bash
cd ~
git clone https://github.com/andrew-verde/ag-chinese-tourism-data.git
cd ag-chinese-tourism-data
```

以后每次开始工作前，先更新到最新代码：

```bash
cd ~/ag-chinese-tourism-data
git pull
```

在 VS Code 中选择 `File` -> `Open Folder`，打开 `ag-chinese-tourism-data` 文件夹。

## 1. 这个项目会生成哪些文件

当前福井小红书分析使用一个手动整理的 Excel 文件和一个分析 CSV：

- `data/raw/social/fukui_xhs_reviews_manual.xlsx`：正式分析使用的小红书手动整理表。每一行是一条小红书笔记，`body_text` 是正文。
- `data/raw/social/fukui_xhs_reviews.csv`：自动脚本抓取的标题候选列表，不是当前正式分析输入。
- `data/processed/fukui_xhs_analysis.csv`：分析后的分类结果，会增加 `theme`、`fan_score`、`travel_score` 等列。

程序现在默认不会直接覆盖旧数据：

- 每次写入已有 CSV 前，会先在 `backups/` 文件夹里保存一个带时间戳的备份。
- 默认会把新抓到的数据和旧 CSV 合并，而不是删除旧数据。
- 如果这次抓取结果为空，程序默认拒绝写入，避免把旧数据清空。
- 如果新结果比旧结果少很多，程序默认拒绝“缩小式覆盖”。

只有在非常确定的情况下，才使用 `--replace --allow-shrink` 这类危险参数。

## 2. 推荐工作方式：VS Code + WSL Ubuntu

### 第一次准备

1. 打开 Windows。
2. 打开 VS Code。
3. 在 VS Code 左下角确认已经连接到 WSL Ubuntu。通常左下角会显示类似 `WSL: Ubuntu`。
4. 用 VS Code 打开这个项目文件夹。
5. 在 VS Code 顶部菜单选择 `Terminal` -> `New Terminal`。
6. 确认终端里显示的是 Ubuntu/Linux，不是 PowerShell。可以输入：

```bash
pwd
```

如果路径类似 `/home/...` 或 `/mnt/c/...`，一般就是 WSL 终端。

### 安装 Python 依赖（第一次，只需做一次）

较新的 Ubuntu 不允许直接用 pip 安装，需要先创建一个"虚拟环境"。在项目文件夹的 WSL 终端里执行：

```bash
sudo apt update
sudo apt install -y python3-venv python3-pip
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
python -m playwright install --with-deps chromium
```

`--with-deps` 会自动安装浏览器需要的系统组件，过程中可能要求输入密码。

### 每次打开新终端时

先激活虚拟环境（终端前面出现 `(.venv)` 字样即成功）：

```bash
cd ~/ag-chinese-tourism-data
source .venv/bin/activate
```

之后文档里的所有 `python3 ...` 命令都在这个状态下运行。

## 3. 运行前如何准备浏览器登录

抓取小红书或抖音需要使用你自己的账号登录。请注意：脚本会打开一个 Playwright 控制的 Chromium 窗口，这个窗口不一定等同于你平时使用的 Chrome。

推荐做法：

1. 先手动打开小红书网页版，确认自己的账号可以正常登录：

```text
https://www.xiaohongshu.com
```

2. 然后运行抓取脚本。脚本会弹出 Chromium 浏览器窗口。
3. 如果弹出的窗口要求登录，请在这个弹出的窗口里登录自己的小红书账号。
4. 登录完成后，如果页面没有继续加载搜索结果，可以关闭脚本并重新运行一次。
5. 抓取过程中不要关闭弹出的 Chromium 窗口。
6. 不建议使用 `--headless`，因为无界面模式下无法手动登录。

如果 WSL Ubuntu 不能弹出浏览器窗口，通常是 WSL 图形界面没有配置好。更简单的解决方法是使用 Windows 版 Python 在 PowerShell 里运行同样命令，或者请熟悉电脑的人帮忙确认 WSLg 是否可用。

## 4. 最简单的一条命令：运行完整福井流程

如果已经有 `data/raw/social/fukui_xhs_reviews_manual.xlsx`，推荐直接运行分析：

```bash
python3 -m src.analysis.xhs_fukui_analysis --input data/raw/social/fukui_xhs_reviews_manual.xlsx --allow-shrink
```

这条命令会读取手动整理的小红书正文，并保存：

```text
data/processed/fukui_xhs_analysis.csv
```

`--allow-shrink` 是因为手动整理表的行数可能少于自动标题候选列表；这是正常的，不代表数据丢失。

如果还没有手动整理表，才先运行抓取候选列表：

```bash
python3 run_fukui_pipeline.py
```

## 5. 分步骤运行

### 只抓取福井小红书数据

旧的自动脚本只抓取搜索结果标题，不进入每条笔记正文。这个脚本保留不变，适合先建立候选列表：

```bash
python3 -m src.scrapers.xhs_fukui_reviews --output data/raw/social/fukui_xhs_reviews.csv
```

增加滚动次数和等待次数：

```bash
python3 -m src.scrapers.xhs_fukui_reviews --output data/raw/social/fukui_xhs_reviews.csv --scrolls 40 --no-growth-limit 6
```

更换或增加搜索关键词，例如只抓取 `福井旅游`：

```bash
python3 -m src.scrapers.xhs_fukui_reviews --keyword "福井旅游" --output data/raw/social/fukui_xhs_reviews.csv
```

同时抓取多个指定关键词：

```bash
python3 -m src.scrapers.xhs_fukui_reviews --keyword "福井旅游,福井美食,永平寺" --output data/raw/social/fukui_xhs_reviews.csv
```

### 手动打开笔记并抓取正文

小红书正文抓取请使用手动辅助脚本。它会打开一个 Playwright Chromium 浏览器，但不会自动点击搜索结果。你需要自己搜索、自己打开笔记；程序只在你按 Enter 后读取当前页面可见的笔记内容。

第一次运行：

```bash
python3 -m src.scrapers.xhs_manual_note_capture --output data/raw/social/fukui_xhs_reviews.csv
```

操作步骤：

1. 浏览器弹出后，在浏览器里登录小红书。
2. 手动搜索关键词，手动点击一条笔记。
3. 确认笔记正文已经显示出来；如果正文被折叠，先手动展开。
4. 回到终端，在 `capture>` 后直接按 Enter。
5. 程序会把当前笔记写入 `data/raw/social/fukui_xhs_reviews.csv`，并按 `note_id` / `note_url` 去重合并。
6. 继续在浏览器里手动打开下一条笔记，再回到终端按 Enter。
7. 结束时输入 `q`，然后按 Enter。

如果只想抓取一条笔记后自动退出：

```bash
python3 -m src.scrapers.xhs_manual_note_capture --once --output data/raw/social/fukui_xhs_reviews.csv
```

脚本会新增或更新这些列：

- `body_text`：当前页面可见的笔记正文。
- `content`：和 `body_text` 相同，方便后续分析使用统一字段名。
- `posted_at`、`captured_at`：页面时间和抓取时间。

如果程序提示没有找到正文，它会直接失败并且不会写入 CSV。常见原因是没有打开具体笔记、正文没有加载出来、正文被折叠，或小红书页面结构变化。

旧的自动标题抓取脚本仍然保留在 `src/scrapers/xhs_fukui_reviews.py`。历史快照保存在 `src/scrapers/history/xhs_fukui_reviews_title_only.py`。

### 只分析已有福井数据

```bash
python3 -m src.analysis.xhs_fukui_analysis --input data/raw/social/fukui_xhs_reviews_manual.xlsx --output data/processed/fukui_xhs_analysis.csv --allow-shrink
```

### 整理手动粘贴的福井抖音评论 Markdown

如果评论是从抖音手动复制到 Word / Google Docs，再导出成 Markdown，请把文件放在：

```text
data/raw/social/fukui_douyin_posts_comments.md
```

然后在仓库根目录运行：

```bash
python3 -m src.analysis.parse_douyin_markdown_comments
```

程序会读取 Markdown，并输出：

```text
data/processed/fukui_douyin_comments_from_md.csv
```

注意：

1. 这个 Markdown 不是抖音官方数据导出，通常没有可靠的作品 ID 或评论 ID。
2. 程序不会猜作品 ID；`douyin_post_id` 会先留空。
3. 每一行 CSV 是一条可识别的评论，适合后续做情感分析。
4. CSV 会保留原始行号和原始文本片段，方便之后检查解析是否正确。
5. 如果重新解析后行数变少，程序会拒绝覆盖旧文件。确认源文件格式变化后，再使用：

```bash
python3 -m src.analysis.parse_douyin_markdown_comments --allow-shrink
```

### 合并小红书和抖音，生成正式分析用数据表

小红书正文和抖音评论不要混在同一个 `body_text` 字段里。原因是：小红书
`body_text` 是笔记作者写的正文；抖音这里解析出来的是评论区内容，属于观众评论。

正式分析前，请运行：

```bash
python3 -m src.analysis.xhs_fukui_analysis --input data/raw/social/fukui_xhs_reviews_manual.xlsx --allow-shrink
python3 -m src.analysis.build_chinese_social_run_dataset --allow-shrink
```

程序会输出两个文件：

```text
data/processed/chinese_social_run_data.csv
data/processed/chinese_social_run_manifest.csv
```

`chinese_social_run_data.csv` 是正式运行用的数据表：

1. 每一行还是一条真实数据：小红书是一条笔记，抖音是一条评论。
2. `platform` 会标明来源是 `xiaohongshu` 还是 `douyin`。
3. `unit_type` 会标明单位是 `note` 还是 `comment`。
4. `text_for_analysis` 是后续统一分析时使用的文本列。
5. 原来的 `title`、`body_text`、`comment_text` 仍然保留，方便检查来源。

`chinese_social_run_manifest.csv` 用来检查本次运行包括了哪些输入。里面应该能看到：

```text
xiaohongshu_analysis
douyin_comments
reviewed_codebook
combined_run_data
```

## 6. 数据安全规则

为了避免再次发生数据丢失，请遵守以下规则：

1. 每次正式抓取前，先确认当前文件夹里已有的 CSV 文件是否还在。
2. 不要手动删除 `data/raw/social/fukui_xhs_reviews.csv`、`data/processed/fukui_xhs_analysis.csv` 或 `backups/` 文件夹。
3. 正常运行时不要使用 `--replace`。
4. 如果程序提示拒绝写入空数据，这是保护机制，不是错误。通常说明没有登录成功、网页结构变了、网络失败，或搜索结果没有加载出来。
5. 如果程序提示拒绝把旧数据替换成更少的行数，不要立刻强制覆盖。先检查新数据是否真的完整。
6. 每次写入旧文件前，程序会自动生成备份，位置类似：

```text
backups/data/raw/social/fukui_xhs_reviews.csv.20260611T120000Z.bak
```

如果误操作，可以从 `backups/` 找回之前的数据。

## 7. 危险参数：只有确认后才使用

正常研究流程不需要下面这些参数。

如果你确实想完全替换旧 CSV：

```bash
python3 -m src.scrapers.xhs_fukui_reviews --replace --allow-shrink --output data/raw/social/fukui_xhs_reviews.csv
```

如果你确实想允许写入空文件：

```bash
python3 -m src.scrapers.xhs_fukui_reviews --allow-empty --output data/raw/social/fukui_xhs_reviews.csv
```

除非你非常确定，否则不要使用这些命令。

## 8. 常见问题

### 问题：浏览器没有弹出来

可能原因是 WSL Ubuntu 的图形界面不可用。可以先确认自己是否使用 Windows 11 的 WSLg。也可以改用 Windows PowerShell 和 Windows 版 Python 运行。

### 问题：显示没有找到任何笔记

常见原因：

- 没有在弹出的 Chromium 窗口里登录小红书。
- 页面加载太慢。
- 小红书要求验证。
- 网页结构发生变化，脚本需要更新。

可以先不要删除任何 CSV，重新登录后再运行：

```bash
python3 run_fukui_pipeline.py
```

### 问题：程序拒绝写入，因为结果为空

这是正常保护机制。旧数据没有被覆盖。请检查浏览器登录状态和网络。

### 问题：程序生成了很多备份文件

这是正常的。备份文件是为了防止学术数据丢失。确认最终数据没有问题后，可以把备份文件复制到安全位置长期保存。

### 问题：怎样确认结果文件有数据

可以运行：

```bash
python3 - <<'PY'
import csv
for path in ["data/raw/social/fukui_xhs_reviews.csv", "data/processed/fukui_xhs_analysis.csv"]:
    try:
        with open(path, encoding="utf-8", newline="") as f:
            rows = list(csv.DictReader(f))
        print(path, "行数:", len(rows))
    except FileNotFoundError:
        print(path, "不存在")
PY
```

## 9. 推荐的正式抓取流程

每次正式抓取建议按这个顺序：

1. 打开 VS Code。
2. 进入 WSL Ubuntu 终端。
3. 确认项目文件夹正确。
4. 激活虚拟环境并执行：

```bash
source .venv/bin/activate
python3 run_fukui_pipeline.py --scrolls 30
```

5. 如果弹出 Chromium 要求登录，在弹出的浏览器中登录小红书。
6. 如果第一次登录后没有抓到数据，重新执行同一条命令。
7. 看到 `Saved ... total rows` 和 `Classification output saved` 后，说明流程完成。
8. 不要删除 CSV 和 `backups/`。

## 数据规则 (Data rules)

1. **分析只使用采集到的真实数据。** 测试用的合成数据只放在 `tests/fixtures/` 目录，
   分析代码（`src/data_loading.py`）不会读取该目录下的文件。
2. **空文件不会被分析。** 如果 CSV 只有表头没有数据行，说明抓取尚未完成，
   分析脚本会提示错误并停止。
3. **所有脚本从仓库根目录运行**，例如 `python3 -m src.scrapers.xhs_fukui_reviews`。
4. 详细背景见 `docs/adr/0001` 与 `docs/adr/0002`。
